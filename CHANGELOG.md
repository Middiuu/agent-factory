# Changelog

Le modifiche pubbliche rilevanti di agent-factory sono raccolte qui. Il progetto non è un package runtime e non applica ancora versioni SemVer: il commit Git resta l'identificatore operativo.

## Unreleased

### Added

- Evidenza eval appaiata, esplorativa e hash-verificabile con transcript, grading, benchmark e viewer statico offline.
- Link checker locale, smoke discovery live schedulato, issue form, template PR, CODEOWNERS, `CONTRIBUTING.md`, `SECURITY.md`, `CODE_OF_CONDUCT.md` e questo changelog.

### Changed

- CI estesa a qualità statica e matrice Ubuntu/macOS con action SHA-pinned.
- Discovery npm corretta per query multi-parola e lookup esatti non applicabili.
- Il template dei report di update apre ora con un executive summary verificabile.
- Esempi rivalidati con evidenze live o limiti osservati, senza promozioni artificiali.

### Security

- Dependabot, segnalazioni private, merge squash-only e cancellazione automatica dei branch abilitati sul remoto.

## 2026-07-10 — nuova linea pubblica

### Added

- Contratto minimale per generare e aggiornare workspace agentici provider-agnostic.
- Blueprint indicizzati per research, web development, mobile development, automation e fallback minimale.
- Discovery bounded, validator strutturali e semantici, fixture positive e negative e CI GitHub Actions.
- Ledger append-only, soglie derivate e governance con approvazione esplicita.

### Changed

- Le skill locali sono opzionali quando capacità native verificate coprono il task.
- Report, privacy, riferimenti confinati, secret scanning e safety web-facing sono stati irrigiditi.
- Il repository remoto è stato ricreato con una nuova `main` originata dal commit radice `5b4ea50`.

### Security

- La nuova linea pubblica esclude la precedente history dall'origin ricreato.
- Fork, cache, mirror o cloni autonomi preesistenti restano fuori dal controllo del progetto.
